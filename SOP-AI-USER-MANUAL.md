## 1. 功能概览

SOP AI 助手提供三大核心功能，贯穿 SOP 的**创建 → 执行 → 发布**全生命周期：

```
┌──────────────────────────────────────────────────────┐
│                  SOP 全生命周期                       │
│                                                      │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐         │
│   │ 创建 SOP │ → │ 执行 SOP │ → │ 发布 SOP │         │
│   └────┬────┘    └────┬────┘    └────┬────┘         │
│        │              │              │                │
│        ▼              ▼              ▼                │
│   ┌─────────┐    ┌─────────┐  ┌──────────┐         │
│   │ AI 生成  │    │ AI 指导  │  │ AI 审核  │         │
│   │ 步骤结构 │    │ 实时建议 │  │ 质量评分 │         │
│   └─────────┘    └─────────┘  └──────────┘         │
└──────────────────────────────────────────────────────┘
```

| 功能 | 场景 | 价值 |
|------|------|------|
| **AI 创建 SOP** | 从零起草，描述需求后 AI 自动生成步骤结构 | 降低写作门槛，快速搭建初稿 |
| **AI 执行指导** | 正在执行某个步骤时，获取 AI 实时建议 | 减少执行错误，提升完成质量 |
| **AI 发布审核** | SOP 发布前，AI 检查完整性、合理性、风险点 | 提升 SOP 质量，避免低级错误 |

**支持模型**：DeepSeek / 智谱 GLM / MiniMax（由用户在配置中自行选择）

## 2. 入口与界面

### 2.1 入口位置

在 **SOP 编辑器页面**（新建或编辑页），工具栏右侧有一个紫色的 **🤖 AI** 按钮。

```
┌─────────────────────────────────────────────────────────────────┐
│  ← 返回    [输入 SOP 标题...]           [📋版本] [🗑删除] [🤖AI] [保存草稿] [发布SOP] │
└─────────────────────────────────────────────────────────────────┘
```

> **Tip**：🤖 AI 按钮在新建 SOP 和编辑已有 SOP 时均可使用。

### 2.2 打开 AI 面板

点击 **🤖 AI** 按钮，面板在编辑器下方展开：

```
┌─────────────────────────────────────────────────────────────────┐
│  ← 返回    [输入 SOP 标题...]           ... [🤖AI] [保存草稿]    │
├─────────────────────────────────────────────────────────────────┤
│  [编辑器主体：标题/描述/步骤列表/定时任务...]                      │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 🤖 SOP AI 助手                              [✕ 关闭]    │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │ [✨AI创建] [💡执行指导] [🔍AI审核] [⚙️模型配置]         │   │
│  ├─────────────────────────────────────────────────────────┤   │
│  │                                                         │   │

│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 四个 Tab 说明

| Tab | 用途 |
|-----|------|
| ✨ AI 创建 | 用 AI 从零生成 SOP 结构 |
| 💡 执行指导 | 步骤执行时获取 AI 实时建议 |
| 🔍 AI 审核 | 发布前对 SOP 做质量审查 |

## 3. AI 创建 SOP

### 3.1 适用场景

- 从零起草一个 SOP，不知道如何拆解步骤
- 有一个模糊想法，想快速得到一个可修改的初稿
- 参考行业最佳实践，让 AI 给出标准流程

### 3.2 操作步骤

**Step 1**：切换到 **✨ AI 创建** Tab

**Step 2**：填写「目标描述」（必填）

在文本框中用自然语言描述你的目标，尽量包含：
- 场景或目的（如"每日早会"、"客户上门拜访"）
- 关键要求（如"简洁高效"、"不超过 30 分钟"）
- 期望包含的环节（如"签到、汇报、总结"）

**示例输入：**
```
每日早会流程，要求简洁高效，包含签到、汇报、总结三个环节，
控制在15分钟内，每个环节有明确的检查要点
```

**Step 3**（可选）：补充标题 / 分类 / 标签

- **标题**：不填则由 AI 自动生成
- **分类**：日 SOP / 周 SOP / 月 SOP / 年 SOP
- **标签**：多个标签用逗号分隔

**Step 4**：点击 **✨ AI 生成 SOP**

等待几秒（取决于网络和模型响应速度），AI 返回结构化 JSON 结果：

```
┌─────────────────────────────────────────┐
│ 生成结果：                               │
│                                         │
│ {                                      │
│   "title": "每日早会 SOP",              │
│   "description": "简洁高效的每日...",    │
│   "category": "daily",                  │
│   "tags": ["早会", "日常"],             │
│   "content": [                          │
│     { "title": "签到", ... },           │
│     { "title": "汇报", ... },           │
│     { "title": "总结", ... }            │
│   ]                                     │
│ }                                      │
│                                         │
│ [应用到此 SOP]                           │
└─────────────────────────────────────────┘
```

**Step 5**：点击 **应用到此 SOP**

AI 生成的 SOP 内容会自动填入编辑器的对应字段：
- 标题 → 标题输入框
- 描述 → 描述文本框
- 分类 → 分类下拉框
- 标签 → 标签输入框
- 步骤列表 → 步骤卡片区域

> **注意**：应用后内容将覆盖编辑器现有数据。如有需要，请先保存当前版本。

### 3.3 生成效果说明

AI 生成的内容包含：
- `title` — 步骤标题
- `description` — 步骤详细说明
- `duration` — 预计时长（分钟）
- `checkItems` — 检查项（用于执行打卡确认）

用户可在应用后自行编辑调整。

## 4. AI 执行指导

### 4.1 适用场景

- 正在执行某个步骤，不确定具体操作是否正确
- 步骤中有容易踩坑的细节，需要 AI 提醒
- 需要正向鼓励和确认感

### 4.2 操作步骤

**Step 1**：切换到 **💡 执行指导** Tab

**Step 2**：填写当前步骤信息

| 字段 | 说明 |
|------|------|
| 步骤标题 | 当前正在执行的步骤名称（必填） |
| 步骤说明 | 当前步骤的详细描述 |
| 当前步骤 | 数字，第几步（如"2"） |
| 总步骤数 | 数字，该 SOP 共几步（如"5"） |
| 用户备注 | 选填，执行中的备注或疑问 |

**Step 3**：点击 **💡 获取 AI 指导**

AI 分析步骤内容后，返回实时指导：

```
┌─────────────────────────────────────────┐
│ AI 指导：                               │
│                                         │
│ ✅ 签到环节关键点：                      │
│                                         │
│ 1. 提前 5 分钟开放签到通道               │
│ 2. 确认全员出席，缺勤需报备              │
│ 3. 记录迟到人员，作为后续考核依据        │
│                                         │
│ ⚠️ 常见踩坑提醒：                        │
│ - 不要跳过签到环节直接开会               │
│ - 远程参会者需截图证明在线               │
│                                         │
│ 💪 加油！签到是早会效率的基础保障！      │
└─────────────────────────────────────────┘
```

### 4.3 使用时机建议

建议在以下时机使用执行指导：
- 每次开始一个新步骤前
- 步骤内容复杂、容易出错时
- 执行过程中遇到不确定情况时

## 5. AI 发布审核

### 5.1 适用场景

- SOP 正式发布前，想确认质量是否达标
- 不知道步骤是否完整、顺序是否合理
- 想让 AI 发现潜在风险和遗漏

### 5.2 操作步骤

**Step 1**：切换到 **🔍 AI 审核** Tab

**Step 2**（可选）：先在编辑器中填写好 SOP 内容

审核基于编辑器中当前保存的 SOP 内容。

**Step 3**：点击 **🔍 AI 审核此 SOP**

AI 对 SOP 进行全面分析，返回结构化审核报告：

```
┌─────────────────────────────────────────┐
│  ✅ 通过                                 │
│  质量评分：85/100                        │
│                                         │
│  问题列表：                              │
│  ⚠️ [unclear] 步骤3描述过于简略         │
│                                         │
│  改进建议：                              │
│  💡 步骤3：建议补充具体操作动作           │
│  💡 步骤5：建议标注预计时长              │
│                                         │
│  审核耗时：1234ms | 模型：deepseek       │
└─────────────────────────────────────────┘
```

### 5.3 审核结论说明

| verdict | 含义 | 建议操作 |
|---------|------|---------|
| `pass` ✅ | 质量合格，可以直接发布 | 可直接点击「发布 SOP」 |
| `warning` ⚠️ |有小问题，建议修复 | 根据建议修改后再发布 |
| `reject` ❌ | 有严重问题，必须修复 | 根据问题列表修改后再审核 |

**审核维度**：
1. **完整性** — 步骤是否齐全、描述是否清晰
2. **合理性** — 步骤顺序是否合理、时长估计是否准确
3. **可执行性** — 检查项是否具体可操作
4. **风险点** — 是否有遗漏的关键步骤或危险操作

## 6. 模型配置

### 6.1 适用场景

- 切换使用不同的 AI 模型（DeepSeek / GLM / MiniMax）
- 配置自己的 API Key（不同模型需要对应平台的 Key）
- 自定义系统提示词或调整温度参数

### 6.2 支持的模型

| 模型 | 提供商 | 默认 API 地址 |
|------|--------|-------------|
| DeepSeek | 深度求索 | `https://api.deepseek.com/chat/completions` |
| 智谱 GLM | 智谱华章 | `https://open.bigmodel.cn/api/paas/v4/chat/completions` |
| MiniMax | MiniMax | `https://api.minimax.chat/v1/text/chatcompletion_v2` |

### 6.3 操作步骤

**Step 1**：切换到 **⚙️ 模型配置** Tab

**Step 2**：填写配置信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 模型类型 | ✅ | 选择 DeepSeek / 智谱 GLM / MiniMax |
| 模型名称 | ❌ | 如 `deepseek-chat`，不填使用默认值 |
| API 地址 | ❌ | 不填使用表格中的默认地址 |
| API Key | ✅ | 在对应 AI 平台申请，不填则使用全局配置 |
| 温度参数 | ❌ | 0.0~2.0，默认 0.7；越低越确定性，越高越创造性 |

**Step 3**：点击 **保存配置**

配置保存在当前用户账户下，后续所有 AI 功能默认使用此配置。

如需清除自定义配置，恢复系统默认，点击 **删除配置**。

### 6.4 获取 API Key

**DeepSeek**：
1. 访问 https://platform.deepseek.com
2. 注册账号并实名认证
3. 在「API Keys」中创建 Key
4. 充值（按 token 计费）

**智谱 GLM**：
1. 访问 https://open.bigmodel.cn
2. 注册并完成认证
3. 在控制台创建 API Key

**MiniMax**：
1. 访问 https://www.minimax.chat
2. 注册并认证
3. 在控制台获取 API Key

## 7. 常见问题

### Q1: AI 生成的内容不满意怎么办？

A：AI 生成结果仅作参考，应用到编辑器后可以：
- 手动增删改步骤
- 调整步骤顺序（拖拽）
- 修改每个步骤的描述和时长
- 再次点击「AI 生成」重新生成

### Q2: API Key 安全吗？

A：API Key 存储在数据库中，仅用于调用对应 AI 平台的接口，不会用于其他用途。建议：
- 不要在公开场合分享你的 Key
- 定期更换 Key
- 为不同平台使用不同的 Key

### Q3: AI 审核"不通过"是不是就不能用了？

A：不是。`reject` 只是表示存在较严重的问题，建议修复后再发布。如果确认内容无误，也可以直接发布，审核结果仅供参考。

### Q4: 没有配置 API Key 能用吗？

A：需要配置 API Key 才能正常调用 AI 功能。默认内置了一个占位 Key（无效），请在「⚙️ 模型配置」中填入真实 Key。

### Q5: 调用 AI 失败怎么办？

A：常见原因：
- API Key 无效或余额不足 → 检查 Key 和账户余额
- 网络超时 → 重试
- 模型服务不可用 → 切换到其他模型

错误信息会显示在对应 Tab 的红色提示框中。

### Q6: 支持微信小程序和 App 吗？

A：当前版本 AI 功能主要面向 PC 端。移动端已接入「AI 执行指导」接口，其他功能可后续按需扩展。
